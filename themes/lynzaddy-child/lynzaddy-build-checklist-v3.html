Understand Section 52 — the color inheritance system
Section 52 is the last block in style.css. It does two critical things:
📖 How Section 52 works
1. Color context lists. Every dark section (lynz-hero, lynz-footer, lynz-cta, etc.) is listed with color: var(--lynz-white). Every light section is listed with color: var(--lynz-midnight). This sets the inherited color at the container level.

2. Inheritance chain. The rule [class*="lynz-"] .elementor-widget-container h1 (and h2–h6) forces all headings inside any lynz-* container to color: inherit. That three-level selector chain is deeper than anything reset.css can produce, so it always wins.

Rule going forward: When you add a new dark section, add its class to the dark list in Section 52. Children inherit white automatically. You will never need to set color on individual headings.

Phase 05

Elementor Flexbox
Containers 101

Read this entire phase before touching the header. The button row pattern is new in v3 — it answers the most common layout question when working with Flexbox Containers.

Everything is a Container
Row Container = the "section"
A Container set to Direction: Row lays its children side by side. The outermost Container on every page section is a Row. Set it to Full Width so the background bleeds edge-to-edge, then Content Width: Boxed 1200px to constrain the children. Add Container → Direction: Row → Width: Full → Content Width: Boxed → CSS class: lynz-hero
Column Container = the "column"
A Container set to Direction: Column stacks its children vertically. Nest it inside a Row Container to create a column in your layout. Set its width by percentage (e.g. 55%) in Layout tab. Nested container inside Row → Direction: Column → Width: 55% → add widgets inside
Applying CSS classes — same for Containers and Widgets
CSS Classes field: Advanced tab → CSS Classes, on any Container or Widget
Outer Row Container (the "section"):
  Click Container handle → Advanced tab → CSS Classes → lynz-hero

Inner Column Container (a "column"):
  Click column container handle → Advanced tab → CSS Classes → lynz-step-left

Widget:
  Click widget → Advanced tab → CSS Classes → lynz-btn-primary

Multiple classes — space-separated:
  lynz-reveal d2

No dot. No quotes. Just the class name.
The button row — getting two buttons side by side
Two buttons side by side = a nested ROW Container holding two Button widgets
In Flexbox Containers, a Column Container stacks everything vertically. Dropping two Button widgets directly into a Column Container will stack them top-to-bottom. To get them side by side, nest a small ROW Container inside the Column Container and place both buttons inside it.

📖 Why this works — and the Wrap setting
The nested ROW Container's Direction: Row places the two Button widgets side by side. Setting Wrap: Wrap (in the Layout tab) means on small screens the second button drops below the first instead of overflowing. Set the ROW Container's width to fit-content or leave at auto so it doesn't stretch full width. You do not need Column Containers inside the button row — Button widgets are flex children directly.
ℹ️This same pattern applies anywhere you need two or more elements side by side inside a Column Container — process step pills, stat pairs, badge + label combos. When something needs to be horizontal inside a column, nest a Row Container.
Full-width background + contained content
How every section achieves full-bleed background + 1200px centered content
Outer Row Container → Layout tab:
  Content Width: Full Width     ← background bleeds edge-to-edge
  Min Height: as needed

Outer Row Container → Style tab:
  Background Type: Classic
  Color: #0D1B2A               ← your section background

Outer Row Container → Advanced tab:
  CSS Classes: lynz-hero         ← handles padding, extra styling

Inner Column Containers:
  Elementor constrains these to 1200px automatically
  because the outer Container is set to Full Width
  with the global Content Width of 1200px.

📖 Why lynz-header, not lynz-nav
CSS selectors like .lynz-nav .elementor-nav-menu a need lynz-nav to actually exist in the DOM. It never does — Elementor doesn't add it. The class you apply is lynz-header on the outer Container.
